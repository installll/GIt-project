# Role: 全栈开发工程师
# Task: 开发一个“智能去水印”Web应用，要求PC和手机端完美适配。

## 技术栈要求：
- 后端：Python Flask
- 前端：原生HTML + CSS (必须使用Tailwind CSS框架实现响应式) + Vanilla JavaScript
- 图像处理库：OpenCV-Python (cv2) 和 NumPy

## 核心功能逻辑 (必须严格遵循)：
1. **上传与显示**：用户点击上传按钮选择图片，图片显示在前端Canvas画布中，并自适应画布大小（不拉伸变形）。
2. **框选水印（支持触控）**：
   - 用户在画布上通过 **鼠标拖拽（PC端）** 或 **手指滑动（移动端）** 绘制一个矩形框，精准框选出水印区域。
   - 必须同时监听 `mousedown/mousemove/mouseup` 和 `touchstart/touchmove/touchend` 事件。
3. **后端处理逻辑**：
   - 前端将 **图片Base64数据** 和 **矩形坐标 (x, y, w, h)** 发送给 Flask 后端。
   - 后端解码图片，根据坐标裁剪出ROI区域。
   - 使用 `cv2.inpaint` 算法（采用 `INPAINT_TELEA` 方法）对该区域进行修复填充。
   - 将修复后的图片转为Base64返回前端。
4. **下载结果**：前端展示去水印后的图片，并提供“下载图片”按钮。

## UI/UX 响应式设计要求：
- **移动端 (手机)**：采用上下垂直布局。顶部是操作工具栏（上传、去水印、下载按钮要大且间距宽，方便触控），中间是图片预览区。
- **PC端 (桌面)**：采用左右布局。左侧是控制面板，右侧是图片展示区。
- 画布宽度必须跟随容器宽度变化（`width: 100%`），高度按比例缩放。
